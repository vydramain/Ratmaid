# Математика плавного хода тележки

Шпаргалка по математике, которая используется в `trash_block.gd`.
Здесь собраны все понятия и формулы из обсуждения, чтобы можно было
вернуться и перечитать.

---

## 1. Две системы координат

У каждого `Node2D` есть **локальная** система координат, и помимо неё
есть **мировая** (общая для всей сцены).

Локальная система тележки определяется её полями:
- `position` — где центр локали стоит в мире;
- `rotation` — на какой угол повёрнуты локальные оси относительно мировых;
- `scale` — масштаб (обычно 1).

При `rotation = 0` локальные оси совпадают с мировыми. При других значениях
оси повёрнуты на этот угол.

В Godot 2D мировая ось Y направлена **вниз** экрана. Положительный
поворот выглядит как «по часовой стрелке».

## 2. Векторы: базовые операции

### Сложение и вычитание

Покомпонентно:
```
(a, b) + (c, d) = (a + c, b + d)
(a, b) - (c, d) = (a - c, b - d)
```

Геометрический смысл:
- `A + v` (точка + вектор) → новая точка, смещённая на `v`.
- `B - A` (точка − точка) → вектор «из A в B». Этим вектором надо
  смещать A, чтобы попасть в B.

### Умножение на скаляр (гомотетия)

```
k · (a, b) = (k·a, k·b)
```

Направление сохраняется (или разворачивается, если `k < 0`), длина
умножается на `|k|`.

### Длина (норма)

Теорема Пифагора:
```
||v|| = sqrt(v.x² + v.y²)
```

В Godot — `v.length()`.

### Нормирование

Деление на собственную длину даёт вектор той же ориентации длиной 1:
```
v / ||v||
```

В Godot — `v.normalized()`. Безопаснее, чем `v / v.length()`,
для нулевого вектора.

### Угол вектора

```
v.angle()  ⇔  atan2(v.y, v.x)
```

Возвращает угол вектора от мировой оси +X, в радианах, в диапазоне
`(-π, π]`. Используется, когда надо «измерить ориентацию» вектора.

### Поворот вектора

`v.rotated(α)` поворачивает вектор `v` на угол `α` и возвращает
новый вектор (не мутирует исходный).

Под капотом — матрица поворота:
```
R(α) = | cos α   -sin α |
       | sin α    cos α |

v.rotated(α) = ( v.x·cos α − v.y·sin α,
                 v.x·sin α + v.y·cos α )
```

## 3. Преобразования между локалью и миром

`Node2D.to_global(p_local)` переводит локальную точку в мировую:
```
p_world = R(rotation) · p_local + position
```

`Node2D.to_local(p_world)` — обратное преобразование:
```
p_local = R(-rotation) · (p_world - position)
```

**Внимание к узлу**, который вызывает функцию: `node.to_local(p)` работает
с локалью именно этого `node`. Если ты хочешь «где точка `p` в локали
тележки», вызывай `trash_block.to_local(p)`, а не `player.to_local(p)`.

## 4. Углы и ориентация

### Поле `rotation`

`rotation` у `Node2D` — обычный `float`, угол в радианах. Не имеет
ограничений (`5π` валидно, `-100` валидно), но геометрически работает
по модулю `2π`.

### Композиция поворотов

Поворот — групповая операция со сложением углов:
```
R(α) · R(β) = R(α + β)
```

То есть «повернуть на α, потом на β» = «повернуть на α + β».

Следствие: если локальный вектор имеет внутри локали угол `φ`, а сама
локаль повёрнута на `rotation`, то в мире у того же вектора угол:
```
world_angle = rotation + φ
```

### `rotation =` vs `rotate()`

- `self.rotation = X` — **присваивание** абсолютного значения. Сразу
  устанавливает поворот в `X`.
- `self.rotate(X)` — **мутация**: прибавляет `X` к текущему `rotation`.
  Это «довернуть ещё на `X`».

Перепутать их = баг. Если у тебя цель — абсолютный угол, используй `=`.

## 5. Плавный поворот к цели

### Целевой угол

Хотим: чтобы вектор `front_direction` в мире смотрел на игрока.
- Текущий мировой угол этого вектора: `rotation + front_direction.angle()`.
- Желаемый мировой угол: `to_player.angle()`.
- Решаем уравнение `rotation + front_direction.angle() = to_player.angle()`:
  ```
  target = to_player.angle() - front_direction.angle()
  ```

Это **абсолютное** значение, которое мы хотим присвоить `rotation`.

### Линейная интерполяция

Для двух чисел `a, b` и параметра `t ∈ [0, 1]`:
```
lerp(a, b, t) = a + t · (b − a) = (1 − t)·a + t·b
```

При `t = 0` результат — `a`, при `t = 1` — `b`.

### Интерполяция углов: `lerp_angle`

Простой `lerp` ломается на углах вблизи ±π (длинная дорога вместо
короткой через границу). `lerp_angle` сначала сводит разность к
короткому пути, потом линейно интерполирует:
```
δ = wrap_to_pi(b − a)        # δ ∈ (-π, π]
lerp_angle(a, b, t) = a + t·δ
```

`wrap_to_pi(x) = x − 2π·floor((x + π) / (2π))`.

### Экспоненциальное затухание

Если каждый кадр пишем `a = lerp_angle(a, target, t)` с постоянным `t`
и фиксированной целью, расстояние `b − a` после каждого кадра
множится на `(1 − t)`. После `N` кадров остаток равен `gap₀ · (1−t)ᴺ`
— геометрическая прогрессия, в непрерывном пределе экспонента.

### Связь с `ROT_SPEED` и `delta`

В коде:
```
rotation = lerp_angle(rotation, target, ROT_SPEED * delta)
```

- `delta` — секунды между кадрами (Godot передаёт это в `_physics_process`).
- `ROT_SPEED` — обратное время релаксации, единицы `1/sec`. Это **не**
  «макс. угловая скорость» — это коэффициент крутизны экспоненты.
- `ROT_SPEED * delta` — безразмерная доля «съесть за один кадр».

За время `T` остаток множится на `exp(-ROT_SPEED · T)`.

Примеры:
- `ROT_SPEED = 8`: за 0.5 сек остаётся `exp(-4) ≈ 1.8%` — быстро.
- `ROT_SPEED = π/2 ≈ 1.57`: за 0.5 сек остаётся `exp(-0.785) ≈ 46%` — лениво.

## 6. Тригонометрия

### Тангенс

Для прямоугольного треугольника с углом `α`:
```
tan(α) = противолежащий / прилежащий = y / x
```

Из подобия треугольников: при заданном `α` это отношение одинаково
для любого размера треугольника. Значит, `tan(α)` — функция от
**только** `α`.

Через синус и косинус:
```
tan(α) = sin(α) / cos(α)
```

### Арктангенс

Обратная функция: по числу даёт угол.
```
arctan(t) = α, при котором tan(α) = t
```

Диапазон: `(-π/2, π/2)`. Это **половина** полного круга — не различает
векторы, направленные в противоположные стороны (`(1, 1)` и `(-1, -1)`
имеют одинаковое `y/x = 1`).

### `atan2(y, x)`

Принимает `y` и `x` по отдельности, поэтому различает все четыре
четверти. Возвращает угол в `(-π, π]`:

```
            arctan(y/x)        если x > 0
            arctan(y/x) + π    если x < 0 и y ≥ 0
atan2(y,x) = arctan(y/x) − π    если x < 0 и y < 0
            +π/2               если x = 0 и y > 0
            −π/2               если x = 0 и y < 0
            не определена      если x = 0 и y = 0
```

`Vector2.angle()` — это `atan2(v.y, v.x)`.

## 7. Ряд Тейлора

### Идея

Любую гладкую функцию рядом с точкой можно приблизить многочленом.
Чем больше степеней — тем точнее.

Рядом с нулём:
```
f(x) ≈ f(0) + f'(0)·x + f''(0)/2! · x² + f'''(0)/3! · x³ + ...
```

`f'(0)` — производная в нуле, и так далее. `n!` — факториал
(`5! = 120`).

В **бесконечном** пределе для «хороших» функций это уже не
приближение, а точное равенство.

### Синус и косинус

Производные синуса в нуле: 0, 1, 0, −1, 0, 1, ... (цикл по 4).

```
sin(x) = x - x³/3! + x⁵/5! - x⁷/7! + ...
cos(x) = 1 - x²/2! + x⁴/4! - x⁶/6! + ...
```

Для `x = 0.5` хватает 3–4 членов, чтобы получить 5 знаков после запятой.

### Зачем

- Калькуляторы и `tan()` в языках считают синус/косинус через ряды
  (или более хитрые алгоритмы вроде CORDIC, основанные на той же идее).
- В играх иногда используют 2–3 члена Тейлора как быструю аппроксимацию,
  если полный `sin`/`cos` дорог.

## 8. Шпаргалка операций Godot

| Запись                                      | Что делает                                          | Закон                              |
|---------------------------------------------|-----------------------------------------------------|------------------------------------|
| `a + b` (векторы)                           | покомпонентная сумма; голова к хвосту               | сложение векторов                  |
| `a - b` (векторы)                           | вектор «из b в a»                                   | сложение с противоположным         |
| `k * v`                                     | масштабирование                                     | гомотетия                          |
| `v.length()`                                | `sqrt(x² + y²)`                                     | теорема Пифагора                   |
| `v.normalized()`                            | вектор длиной 1, направление сохранено              | деление на норму                   |
| `v.angle()`                                 | `atan2(v.y, v.x)`, угол от мировой +X, ∈ (-π, π]   | определение полярного угла         |
| `v.rotated(α)`                              | поворот вектора на α                                | матрица поворота R(α)              |
| `node.to_local(p_world)`                    | мировая точка → локаль `node`                       | обратное аффинное преобразование   |
| `node.to_global(p_local)`                   | локаль `node` → мировая точка                       | прямое аффинное преобразование     |
| `lerp(a, b, t)`                             | линейная интерполяция чисел                         | `a + t·(b − a)`                    |
| `lerp_angle(a, b, t)`                       | интерполяция углов по кратчайшей дуге               | `a + t·wrap_to_pi(b − a)`          |
| `rotation = X`                              | присвоить абсолютный угол                           | —                                  |
| `rotate(X)`                                 | прибавить дельту к `rotation`                       | —                                  |
| `move_and_slide()`                          | продвинуть `CharacterBody2D` на `velocity * delta` с обработкой коллизий | физика твёрдого тела |
